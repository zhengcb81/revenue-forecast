# Revenue Forecast v2.5.0 - 快速入门指南

## 🎯 5分钟上手

### 第一步：启动分析

在Claude Code中输入：
```
使用revenue-forecast skill分析[公司名称]
```

例如：
```
使用revenue-forecast skill分析小米集团
```

### 第二步：跟随系统提示

系统会自动执行11步分析流程，你只需：
1. **确认公司类型** - 系统自动识别，如有误请指正
2. **回答关键问题** - 如"是否有品牌矩阵？"
3. **审核关键输出** - 重点关注Step4的9维度分析

### 第三步：获取结果

分析完成后，在`outputs/`目录查看：
- `RevGrowth_小米集团.json` - 结构化数据
- `RevGrowth_FullReport_小米集团.md` - 完整报告

## 📊 按公司类型的分析重点

### 产品驱动型 (如：小米、苹果、茅台)
**关键分析点**：
- 品牌矩阵分析（Step4.5）
- 产品线扩张策略
- 市场份额变化
- 定价能力

**期望输出**：
- 品牌健康度评分
- 各产品线增长预测
- 整体CAGR和投资评级

---

### 基础设施驱动型 ⭐v2.5 (如：长江电力、新奥能源)
**关键分析点**：
- 监管资产基数（RAB）
- 准许收益率（8-10%）
- 用户渗透率
- 价格调整机制

**期望输出**：
- RAB增长预测
- 监管框架影响评估
- 稳定分红能力分析

---

### 项目制驱动型 ⭐v2.5 (如：中国建筑、中国船舶)
**关键分析点**：
- 在手订单/营收比（Backlog Ratio）
- 订单获取能力
- 项目交付周期
- 现金回款情况

**期望输出**：
- 订单执行预测
- 周期性风险评估
- 毛利率波动分析

---

### 代工驱动型 (如：台积电、药明康德)
**关键分析点**：
- 产能利用率
- 技术节点突破
- 客户集中度
- 资本开支计划

**期望输出**：
- 产能扩张时间表
- 技术升级影响
- 地缘政治风险评估

---

### 房地产/REITs驱动型 ⭐v2.5 (如：普洛斯、万科)
**关键分析点**：
- Cap Rate（资本化率）
- NOI（净营业收入）
- 出租率
- WALE（加权平均租约期限）

**期望输出**：
- NAV估值
- 分红收益率预测
- 资产增值潜力

## 🔥 v2.5.0 新功能快速使用

### 1. 压力测试
在分析高风险公司时，系统会自动执行压力测试：
```
Step4.5: 压力测试
- 情景1: 地缘政治风险（台海冲突）
- 情景2: 行业政策变化（集采扩围）
- 情景3: 技术路线颠覆
...
输出: 生存概率评估、恢复路径分析
```

### 2. 蒙特卡洛模拟
为关键假设添加概率分布：
```yaml
# 在config.yaml中配置
monte_carlo:
  iterations: 10000
  variables:
    market_growth: {distribution: "normal", mean: 15, std: 5}
    market_share: {distribution: "uniform", min: 10, max: 20}
    price_change: {distribution: "triangular", low: -5, mode: 0, high: 10}
```

**输出**：
- 营收概率分布图
- P10/P25/P50/P75/P90分位数
- VaR（风险价值）
- 敏感性分析（龙卷风图）

### 3. 跨公司比较
分析多家同行业公司：
```
分析行业: 新能源汽车
公司: 比亚迪、蔚来、小鹏、理想
输出: 
- 五维雷达图对比
- 行业排名表
- 投资建议矩阵
```

### 4. ESG调整
评估ESG因素对营收的影响：
```
E(环境): 碳成本 -2%
S(社会): 供应链责任 +1%
G(治理): 股权质押风险 -1%
-------------------------
ESG调整系数: -2%
调整后营收预测: 原预测 × 0.98
```

- [ ] 调用正确的公司类型框架
- [ ] 核心指标计算完整

- [ ] 乐观/基准/悲观三种情景
- [ ] 每种情景有明确假设
- [ ] 概率权重合理

### Step8 报告验证
- [ ] 语言：中文为主（<30%英文）
- [ ] 文件：JSON + Markdown两个文件
- [ ] 章节：6个必需章节齐全
- [ ] 大小：JSON>2KB, Markdown>10KB

## 🛠️ 常见问题速查

| 问题 | 解决方案 |
|-----|---------|
| 公司类型识别错误 | 在Step2人工纠正，参考type-detection.md |
| 某个维度分析太浅 | 要求 deeper analysis，使用WebSearch获取更多数据 |
| 报告生成失败 | 检查validate_report.py输出，修复问题后重试 |
| 缓存冲突 | 删除revenue-forecast-cache/[公司名]/目录重新分析 |
| 想要更保守的预测 | 调整config.yaml中的scenario_probabilities |
