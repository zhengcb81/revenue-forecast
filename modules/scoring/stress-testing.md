# 压力测试分析框架 ⭐️⭐️⭐️ 新增（黑天鹅事件与极端情景分析）

**版本**: v1.0  
**创建日期**: 2026-03-01  
**适用范围**: 所有公司类型，特别是地缘政治敏感行业、政策敏感行业、高杠杆企业

---

## 核心概念

传统情景分析（乐观/基准/悲观）基于**概率分布内的合理波动**，但现实世界可能出现**尾部风险事件**——概率极低但影响极大的"黑天鹅"。压力测试就是对这些极端情景的量化分析。

**与传统情景分析的区别**:

| 维度 | 情景分析 (乐观/基准/悲观) | **压力测试** |
|-----|------------------------|-------------|
| **发生概率** | 25%/50%/25%（高概率） | **<5%（尾部风险）** |
| **影响程度** | ±20%营收波动 | **-50%甚至企业生存危机** |
| **分析目的** | 预测营收区间 | **评估生存能力和恢复路径** |
| **触发条件** | 市场正常波动 | **极端外部冲击** |
| **输出结果** | 概率加权CAGR | **生存概率+恢复时间+应对措施** |

**为什么要做压力测试**:
1. **识别隐藏风险**: 常规分析可能忽视的尾部风险
2. **评估韧性**: 企业在极端冲击下的生存能力
3. **制定预案**: 提前准备应对极端情景的策略
4. **投资者保护**: 避免投资者因忽视尾部风险而受损

```

- 美元债务偿还压力骤增
- 可能触发债务违约
- 典型: 房地产美元债、航空租赁负债
```

### 二、压力测试量化方法

#### 2.1 压力测试参数设定

**压力情景设计原则**:
```
1. 相关性: 压力情景与企业业务相关
2. 极端性: 发生概率<5%，但影响巨大
3. 可量化: 能够将压力传导至财务指标
4. 可恢复: 分析企业从压力中恢复的路径
```

**关键压力参数**:

| 参数类别 | 基准值 | 轻度压力 | 中度压力 | 重度压力 |
|---------|-------|---------|---------|---------|
| **GDP增速** | 5% | 3% | 0% | -3% |
| **行业需求** | 基准 | -20% | -40% | -60% |
| **产品价格** | 基准 | -15% | -30% | -50% |
| **原材料成本** | 基准 | +20% | +40% | +60% |
| **融资成本** | 5% | 8% | 12% | 18% |
| **应收账款回收期** | 60天 | 90天 | 120天 | 180天 |

#### 2.2 压力传导模型

**三层传导模型**:
```
第一层: 外部冲击 → 直接影响
    地缘政治/政策变化 → 需求/价格/成本变化

第二层: 运营影响 → 财务指标
    需求↓/成本↑ → 营收↓/利润↓/现金流↓

第三层: 财务恶化 → 生存风险
    现金流断裂/债务违约 → 企业生存危机
```

**压力测试计算示例（以半导体设备企业为例）**:

```
基准情景（2024年）:
- 营收: 100亿元
- 毛利率: 45%
- 经营现金流: 20亿元
- 现金储备: 50亿元
- 年研发费用: 15亿元

压力情景: 美国进一步制裁，关键零部件断供

直接影响（第一年）:
- 新产品研发停滞，老客户流失
- 营收下降: 100亿 → 40亿元 (-60%)
- 毛利率下降: 45% → 35%（规模效应丧失）
- 净利润: 从15亿 → -10亿元（亏损）

财务恶化（第二年）:
- 经营现金流: 20亿 → -5亿元
- 现金储备: 50亿 → 35亿元（消耗15亿）
- 若压力持续，第三年现金储备降至20亿元

生存评估:
- 现金储备/年研发费用 = 50/15 = 3.3年
- 在重度压力下，企业可维持3年研发投入
- 生存概率: 高（若3年内有替代方案）
- 恢复时间: 5-7年（重新建立供应链）
```

#### 2.3 生存概率评估模型

**生存概率评分体系**:

| 评估维度 | 权重 | 高分标准 | 低分标准 |
|---------|------|---------|---------|
| **现金储备** | 30% | >3年运营费用 | <1年运营费用 |
| **债务结构** | 25% | 长期债务为主，无短期偿付压力 | 短期债务集中到期 |
| **业务多元化** | 20% | 多区域、多客户、多产品线 | 单一市场/客户依赖 |
| **替代方案** | 15% | 有可行的替代供应商/技术路线 | 完全依赖被制裁方 |
| **政府支持** | 10% | 明确的国家战略支持 | 无政策支持 |

**生存概率计算**:
```
生存概率 = Σ(维度得分 × 权重)

- 80-100分: 高概率生存（>80%）
- 60-80分: 中等概率生存（50-80%）
- 40-60分: 低概率生存（20-50%）
- <40分: 生存危机（<20%）
```

#### 3.3 银行业压力测试

**压力情景: 房地产不良贷款率升至15%**

```
压力测试计算:
- 房地产贷款占比: 30%
- 不良贷款率: 15%
- 拨备覆盖率要求: 150%

额外拨备需求 = 总贷款 × 30% × 15% × 150% = 总贷款 × 6.75%

对净利润影响:
- 若银行净利润率1%，年净利润需用于拨备，可能连续2-3年零利润或亏损
- 资本充足率下降，限制信贷扩张能力
- 股价可能下跌50%+

系统性风险:
- 若多家银行同时面临压力，可能引发信贷紧缩
- 实体经济融资困难，经济下滑加剧
```

### 压力情景参数来源

| 参数类别 | 设定依据 | 数据来源 | 更新频率 |
|---------|---------|---------|---------|
| **地缘政治风险** | 国际关系评估、智库报告 | CSIS、IISS等 | 季度 |
| **政策风险** | 监管政策动向、行业反馈 | 监管公告、行业协会 | 实时 |

### 压力传导系数校准

压力传导系数基于历史事件回归分析：
- 需求冲击: 基于2008年、2020年历史数据
- 价格冲击: 基于大宗商品价格波动历史
- 信用紧缩: 基于2018年去杠杆、房地产融资收紧经验

## 总结

压力测试是营收预测框架的重要补充，用于：

1. **识别尾部风险**: 发现常规分析可能忽视的极端风险
2. **评估企业韧性**: 量化企业在极端冲击下的生存能力
3. **指导投资决策**: 帮助投资者理解最坏情况下的损失
4. **完善风险管理**: 为企业制定风险应对预案提供依据

**使用建议**:
- 所有公司都应进行基础压力测试（至少1-2个相关情景）
- 高风险行业（半导体、房地产、银行）应进行全面的多情景压力测试
- 压力测试不应替代情景分析，而是作为补充
- 压力测试结果应定期更新（至少每年一次）

**局限性**:
- 极端情景难以精确预测
- 压力传导机制存在不确定性
- 恢复路径假设可能过于乐观
- 黑天鹅定义就是不可预测，压力测试只能覆盖已知风险

## 压力测试计算过程展示规范 ⭐️ v2.6.0新增

### 核心原则

压力测试章节必须采用**四段落结构**：
1. **假设依据** - 说明情景定义和发生概率
2. **计算过程** - 展示传导机制和财务影响量化
3. **验证检查** - 验证生存能力评估
4. **结论** - 给出生存概率和恢复路径

### 示例模板

```markdown
### 压力测试：台海冲突升级

#### 情景定义

**触发条件**：台海军事冲突升级，西方国家对中国实施全面制裁

**发生概率**：5%（尾部风险事件）

**影响周期**：12-24个月

#### 传导机制分析

**第一层：直接冲击**

| 冲击类型 | 影响机制 | 量化估计 |
|---------|---------|---------|
| 供应链中断 | 台湾芯片供应中断 | 芯片进口-60% |
| 海外市场关闭 | 欧美禁售 | 海外营收-40% |
| 金融市场冲击 | 融资成本+5%，汇率-10% | 财务成本+3% |

**第二层：间接冲击**

| 冲击类型 | 影响机制 | 量化估计 |
|---------|---------|---------|
| 消费信心下降 | 国内消费萎缩 | 国内营收-15% |
| 供应链重构 | 替代供应商切换 | 一次性成本5% |

#### 财务影响量化

**冲击传导计算**：

```
当前营收：1000亿

第一层冲击后 = 1000 × (1-25%) = 750亿（供应链中断）
第二层冲击后 = 750 × (1-40%) = 450亿（海外市场关闭）
第三层冲击后 = 450 × (1-3%) = 436.5亿（财务成本增加）
第四层冲击后 = 436.5 × (1-15%) = 371亿（国内消费萎缩）

营收下降幅度：(1000-371)/1000 = 62.9%
```

#### 生存能力评估

**流动性分析**：

```
现金储备：200亿
月均运营成本：50亿
可支撑月数：200 / 50 = 4个月

银行授信额度：300亿
合计可支撑：(200+300) / 50 = 10个月
```

**生存概率判定**：

| 可支撑月数 | 生存概率 | 本案例 |
|-----------|---------|--------|
| >6个月 | 高(70%+) | ✓ |
| 3-6个月 | 中(40-70%) | |
| <3个月 | 低(<40%) | |

#### 结论

**结论**：
- 营收影响：下降 **62.9%**（从1000亿降至371亿）
- 生存概率：**70%**（可支撑10个月）
- 恢复时间：**24-36个月**

**恢复路径**：
- 0-6个月：危机应对，止损
- 6-12个月：供应链重构
- 12-24个月：市场恢复
- 24-36个月：完全恢复
```
