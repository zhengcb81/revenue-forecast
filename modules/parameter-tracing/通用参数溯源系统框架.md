# 通用参数溯源系统框架 v1.0
**适用于**: revenue-forecast skill 所有公司类型分析
**框架版本**: v2.5.0
**创建日期**: 2026-01-15

---

## 框架目标

为revenue-forecast skill建立统一的参数溯源系统，确保：
1. **通用性**: 适用于13种公司类型（产品/设备/资源/爆发/服务/代工/资本/研发/订阅/平台/基础设施/项目制/房地产驱动型）
2. **标准化**: 所有报告使用相同的溯源格式
3. **可验证性**: 参数来源可追溯、可验证
4. **自动化**: 尽可能自动收集和验证参数数据
5. **可靠性分级**: 参数可信度可评估

```json

  "unit": "单位",
  "data_source": {
    "primary_source": "主要数据来源",
    "source_type": "来源类型",
    "source_url": "数据链接（如适用）",
    "retrieval_date": "获取日期",
    "validation_method": "验证方法",
```

## 四、参数验证机制

### 4.1 自动化验证规则

```python
def validate_parameter(param):
    """参数验证规则"""
    issues = []

    # 规则1: 必填字段检查
    required_fields = [
        "parameter_name", "value", "unit",
        "primary_source", "source_type", "retrieval_date",
        "reliability_score", "confidence_level"
    ]
    for field in required_fields:
        if field not in param:
            issues.append(f"缺少必填字段: {field}")

    # 规则2: 数据类型检查
    if not isinstance(param["value"], (int, float, str)):
        issues.append("value字段类型错误")

    # 规则3: 可靠性评分范围检查
    score = param.get("reliability_score")
    if score is not None and (score < 1 or score > 10):
        issues.append("reliability_score必须在1-10之间")

    # 规则4: 来源类型检查
    valid_types = [
        "company_financials", "company_guidance",
        "regulatory_filing", "press_release",
        "industry_report", "market_data",
        "analyst_report", "news_media",
        "calculated", "estimate", "benchmark", "historical_trend"
    ]
    if param.get("source_type") not in valid_types:
        issues.append(f"无效的source_type: {param.get('source_type')}")

    # 规则5: 日期格式检查
    date_pattern = r"\d{4}-\d{2}-\d{2}"
    if not re.match(date_pattern, param.get("retrieval_date", "")):
        issues.append("retrieval_date格式错误，应为YYYY-MM-DD")

    return issues
```

### 4.2 数据一致性检查

```python
def cross_validate_parameters(params):
    """交叉验证参数一致性"""
    issues = []

    # 规则1: 营收数据一致性
    if "current_revenue" in params and "revenue_by_segment" in params:
        total = sum(params["revenue_by_segment"].values())
        if abs(total - params["current_revenue"]) / params["current_revenue"] > 0.05:
            issues.append("分业务营收与总营收差异超过5%")

    # 规则2: 驱动因素一致性
    if "revenue_drivers" in params and "total_revenue_cagr" in params:
        total_driver_contribution = sum(
            driver["contribution"]
            for driver in params["revenue_drivers"]
        )
        if abs(total_driver_contribution - params["total_revenue_cagr"]) > 2:
            issues.append("驱动因素贡献度与总CAGR差异超过2个百分点")

    return issues
```

### 5.1 更新触发条件

| 参数类型 | 更新频率 | 触发条件 |
|---------|---------|---------|
| **财务数据** | 季度 | 公司发布季度财报 |

| **竞争数据** | 季度 | 竞争对手发布财报 |

### 5.2 自动化更新建议系统

```python
def suggest_update(param, last_update, current_date):
    """建议是否需要更新参数"""

    # 根据参数类型确定更新周期
    update_cycles = {
        "company_financials": 90,      # 90天
        "regulatory_filing": 90,
        "market_data": 30,             # 30天
        "industry_report": 180,         # 180天
        "analyst_report": 60
    }

    source_type = param.get("source_type")
    cycle = update_cycles.get(source_type, 180)

    days_since_update = (current_date - last_update).days

    if days_since_update > cycle:
        return {
            "needs_update": True,
            "reason": f"距离上次更新已超过{cycle}天",
            "priority": "high" if days_since_update > cycle * 2 else "medium"
        }

    return {"needs_update": False}
```
