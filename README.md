---

## 📋 系统概述

Revenue Forecast Skill是一个**零硬编码、完全配置驱动**的营收增长预测分析系统,用于帮助买方分析师快速、准确地预测公司未来5-7年的营收增长。

### 核心特性

✅ **零硬编码**: 所有参数通过`config.yaml`配置化,包括年份、阈值、路径等
✅ **动态年份**: 使用`datetime.now().year`自动计算预测年份,无需每年修改
✅ **13种公司类型**: 全覆盖主流商业模式
✅ **多维度防跳过机制**: Token+内容+工具+文件四维度验证
✅ **蒙特卡洛模拟**: 10,000次随机模拟生成概率分布
✅ **压力测试**: 10个极端情景风险测试
✅ **准确度追踪**: 预测→反馈→校准闭环系统
✅ **情景预测**: 乐观/基准/悲观三种情景,支持多种概率模式
✅ **直接评分**: 基于CAGR直接查表评分,无主观调整
✅ **验证检查点**: 3个检查点确保分析质量
✅ **缓存系统**: 智能缓存搜索结果,提高效率

### 核心组件说明

**1. config.yaml (⭐最重要)**
- 所有配置的单一来源
- 年份、阈值、路径、评分表等全部可配置
- 修改配置即可调整系统行为,无需修改代码

**2. config.py**
- 配置加载器,提供950行配置访问接口
- 动态年份计算、路径构建、验证等功能
- 零硬编码,所有参数从YAML读取

**3. executor.py**
- 执行引擎,强制执行10步分析流程
- 3个验证检查点确保质量
- 缓存系统管理

**4. modules/**
- 32个Markdown文档,包含所有分析框架
- 每个公司类型有专门的分析模块
- 评分框架、报告模板、高级分析模块

**5. enforcement_controller.py (v2.5.0新增)**
- 多维度防跳过验证系统
- Token消耗、内容深度、工具调用、文件生成四维度
- 确保分析质量不受模型速度影响

## ⚙️ 配置指南

### 核心配置项

**1. 年份配置**(动态)
```yaml
forecast:
  start_year: "dynamic"  # 使用当前年份
  end_year_offset: 5      # 预测未来5年
  historical_year_offset: 1  # 历史数据年份
```

**2. 情景概率配置**
```yaml
scenario_probabilities:
  default:  # 默认模式
    optimistic: 0.25
    base: 0.50
    pessimistic: 0.25
  national_strategy:  # 国家战略驱动模式
    optimistic: 0.30
    base: 0.60
    pessimistic: 0.10
```

**3. 评分表配置**
```yaml
scoring_table:
  - cagr_range: [30, 100]
    score_range: [9.5, 10.0]
    rating: "强烈推荐"
  - cagr_range: [25, 30]
    score_range: [8.5, 9.4]
    rating: "推荐"
  # ... 更多评分区间
```

**4. 目录路径配置**
```yaml
directories:
  cache_base_dir: "revenue-forecast-cache"
  output_dir: "outputs"
  search_results_subdir: "search-results"
```

### 修改配置

所有配置都在`core/config.yaml`中,修改后立即生效,无需重启系统。

### v2.5.0新增: 全面升级 - Phase 1-3 完成 🚀

#### 1. 新增4种公司类型 (13种全覆盖)
- **基础设施驱动型**: 电力、水务、燃气、高速公路(RAB监管模型)
- **项目制驱动型**: 建筑、造船、EPC、军工(订单 backlog 分析)
- **平台驱动型**: 电商、社交、内容平台(网络效应分析)
- **房地产/REITs驱动型**: 商业地产、物流地产、REITs(Cap Rate模型)

#### 2. 多维度防跳过机制 (核心改进)
**问题**: 传统时间验证受模型速度影响，不公平  
**解决方案**: 四维度客观验证系统

| 维度 | 衡量指标 | Step4要求 | 说明 |
|-----|---------|----------|------|
| **Token消耗** | 思考深度 | 8000+ | 不受速度影响 |
| **内容产出** | 输出质量 | 5000+字符 | 客观衡量 |
| **工具调用** | 行动次数 | 18+次 | 搜索深度 |
| **文件生成** | 工作成果 | 9个文件 | 维度分析 |

**通过逻辑**: (内容+文件)必须满足 AND (Token+工具)互补

#### 3. 蒙特卡洛模拟
- **10,000次随机模拟**生成营收预测概率分布
- 输入参数概率分布设定
- 输出: P10/P25/P50/P75/P90分位数、VaR风险价值
- 敏感性分析(龙卷风图)

#### 4. 压力测试模块
**10个极端情景**:
- 地缘政治: 台海冲突、中美科技脱钩、俄乌式制裁
- 行业政策: 医药集采扩围、教育式政策、房地产硬着陆
- 金融风险: 2008式信贷危机、汇率剧烈波动
- 技术颠覆: 技术路线颠覆、AI对服务业颠覆

#### 5. 滚动预测与准确度追踪
- **季度更新机制**: Q1→Q2→Q3→Q4 动态调整预测
- **准确度指标**: MAPE、方向准确率、置信区间覆盖率
- **自动校准**: 生成校准系数(如"系统5%乐观，应用0.95因子")

#### 6. ESG风险调整因子
- **E(环境)**: 碳成本、环保合规 → -5%~+5%
- **S(社会)**: 劳工成本、供应链责任 → -3%~+3%
- **G(治理)**: 股权质押、关联交易 → -4%~+4%
- 量化ESG对企业利润的影响

#### 7. AI辅助验证工具
四维自动验证:
- **完整性**: 必需字段、必需章节
- **一致性**: CAGR与评分匹配、情景排序
- **数据质量**: 数据密度、单位一致性
- **可读性**: 结构层次、段落分布

输出: 综合评分(0-100)、质量评级、改进建议

#### 8. 跨公司比较分析
**五维比较框架**:
- 增长能力(30%)、盈利能力(25%)、估值水平(20%)、风险控制(15%)、质量护城河(10%)

输出: 行业排名表、雷达图对比、投资建议矩阵

## 🎯 评分体系

### 综合CAGR直接评分表

| 综合CAGR范围 | 评分 | 投资评级 |
|-------------|------|---------|
| **≥ 30%** | **9.5-10.0** | ✅ 强烈推荐 |
| **25% - 30%** | **8.5-9.4** | ✅ 推荐 |
| **20% - 25%** | **7.5-8.4** | ✅ 推荐 |
| **18% - 20%** | **7.0-7.4** | ☑️ 接近推荐 |
| **15% - 18%** | **6.5-6.9** | ☑️ 适度配置 |
| **12% - 15%** | **5.5-6.4** | ☑️ 适度配置 |
| **10% - 12%** | **4.5-5.4** | ⚠️ 观望 |
| **8% - 10%** | **3.5-4.4** | ⚠️ 观望 |
| **5% - 8%** | **2.5-3.4** | ❌ 不推荐 |
| **< 5%** | **0.5-2.4** | ❌ 不推荐 |

### ⚠️ 严格执行规则

1. **必须严格按照CAGR查表,不得主观调整**
2. **不允许因为公司是龙头、品牌好、市值大而加分**
3. **所有风险、收益因素都已在情景中反映,不得事后调整**

3. 在`type-detection.md`中添加检测规则

### Q4: 为什么年份是动态的?

**A**: 使用`datetime.now().year`自动获取当前年份,这样:
- ✅ 无需每年修改代码
- ✅ 2026年自动预测2026-2031
- ✅ 2027年自动预测2027-2032

### Q5: 缓存系统如何工作?

**A**:
- 缓存目录: `revenue-forecast-cache/[公司名称]/`
- 搜索结果: `search-results/dimension-*.md`
- metadata.json: 记录分析历史和状态
- 下次分析时自动复用缓存,提高效率

### Q6: 多维度防跳过机制如何工作? ⭐v2.5

**A**: 传统时间验证不公平(模型速度不同),v2.5采用四维度客观验证:
- **硬性要求**(必须满足): 内容深度 + 文件生成
- **软性要求**(互补验证): Token消耗 + 工具调用

例如Step4: 必须产出5000+字符内容 + 生成9个维度文件，同时推荐消耗8000+Tokens和调用18+次工具。

### Q7: 如何使用蒙特卡洛模拟? ⭐v2.5

**A**: 在Step4.5执行蒙特卡洛模拟:
1. 定义输入参数的概率分布(市场增长率、市占率、价格等)
2. 系统自动执行10,000次随机模拟
3. 输出营收预测的概率分布(P10/P25/P50/P75/P90)
4. 生成VaR风险价值和敏感性分析(龙卷风图)

### Q8: 如何进行跨公司比较? ⭐v2.5

**A**: 使用跨公司比较框架:
```
1. 选择3-5家同行业公司
2. 五维评分: 增长(30%)、盈利(25%)、估值(20%)、风险(15%)、质量(10%)
3. 生成行业排名表和雷达图对比
4. 输出投资建议矩阵(买入/持有/卖出)
```

### Q9: 预测准确度追踪有什么用? ⭐v2.5

**A**: 准确度追踪建立预测→反馈→校准闭环:
- 记录每次预测的时间和置信区间
- 与实际结果对比计算MAPE、方向准确率
- 生成校准系数(如"系统5%乐观，应用0.95因子")
- 持续改进未来预测准确性

### Q10: ESG调整因子如何影响评分? ⭐v2.5

**A**: ESG三维评估量化对企业利润的影响:
- **E(环境)**: 碳成本、环保合规 → -5%~+5%
- **S(社会)**: 劳工成本、供应链责任 → -3%~+3%
- **G(治理)**: 股权质押、关联交易 → -4%~+4%

调整后的营收 = 原预测 × (1 + ESG调整系数)
