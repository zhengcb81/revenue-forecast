  - 宝马案例分析发现: 电动化业务30%CAGR假设过于乐观
  - 缺乏竞争强度维度，导致预测与现实脱节
  - 未分区域预测，全球一刀切
  - 品牌矩阵与营收预测脱节
  - 技术追赶风险未量化

     - 避免对中国等高竞争市场过度乐观

     - 新增区域预测模板: 中国/欧洲/美国/其他
     - 综合CAGR = Σ(区域权重 × 区域调整后CAGR)
     - 识别结构性机会与风险

  3. **品牌贡献因子量化** (modules/company-types/product-driven.md):
     - 新增品牌贡献因子计算公式
     - 5大因素: 品牌协同(20%)/扩张(15%)/高端化(25%)/新品牌(25%)/溢价(15%)
     - 品牌贡献可直接计入总CAGR

  4. **技术追赶概率模型** (modules/scoring/scenario-analysis.md):
     - 3种情景: 快速(+3%)/缓慢(0%)/持续落后(-5%)
     - 期望值量化技术风险

  **预期效果**:
  - 宝马案例综合CAGR从+2.8%调整至-1%至0%
  - 更贴近现实，避免过度乐观假设

**执行目的**: 在开始分析前加载系统配置,确保缓存目录、输出目录、维度文件等配置正确

**强制执行步骤**:

1. **读取配置模块**

   - 重点阅读: 目录配置、维度文件配置、公司类型配置

2. **验证配置加载**

   # 调用配置验证函数
   from core.config import (
       get_cache_base_dir,
       get_output_dir,
       get_dimension_files,
       get_company_types,
       validate_config
   )

   # 验证配置完整性
   is_valid, errors = validate_config()
   if not is_valid:
       for error in errors:
           print(f"❌ 配置错误: {error}")

   ✅ 已读取: core/config.py

原因: 未读取配置模块 `core/config.yaml`

解决方案:

## v2.5.0 更新：计算过程记录强制要求 ⭐️ 新增

### 核心原则

**⚠️ 重要**：所有关键结论必须展示完整的计算过程，不得直接给出结论数字！

### 四段落结构

每个关键结论章节必须包含以下四个段落：

```markdown
### X.Y.1 假设依据
[说明参数来源、数据依据、判断逻辑]

### X.Y.2 计算过程
[展示公式、代入数值、逐步计算]

### X.Y.3 验证检查
[交叉验证、合理性检查]

### X.Y.4 结论
**结论**：[简洁明确的最终数字]
```

### 必须展示计算过程的章节

| 章节 | 必须展示的内容 |
|-----|---------------|
| 情景预测 | 逐年营收计算、CAGR验证 |
| 综合CAGR | 加权平均计算过程 |
| 评分 | 查表过程、线性插值计算 |
| 蒙特卡洛 | 输入参数分布、统计结果 |
| ESG调整 | E/S/G因子逐项计算 |
| 压力测试 | 传导机制、财务影响量化 |
| 品牌矩阵 | 协同效应评分、贡献度预测 |
| 管线价值 | rNPV贴现计算 |
| 订阅指标 | LTV/CAC、NRR分解 |

### 结论格式规范

**单个数字结论**：
```markdown
**结论**：综合5年CAGR = **14.5%**
```

**多个数字结论**：
```markdown
**结论**：
- 综合5年CAGR = **14.5%**
- 2029年预测营收 = **722.9亿**
- 综合评分 = **6.3分**
```

### JSON输出增强

JSON报告中必须包含 `calculation_trace` 字段，记录所有关键计算的中间过程。

详见：`modules/output/json-template.md`
