## Phase 1: 基础扩展（紧急改进）

### 1.1 新增基础设施/公用事业驱动型 ⭐

**文件**: `modules/company-types/infrastructure-driven.md`

**适用公司**:

- 特许经营权价值评估
- 价格调整机制

**案例覆盖**: 长江电力、新奥能源深度分析

### 1.2 新增项目制/订单驱动型 ⭐

**文件**: `modules/company-types/project-driven.md`

**适用公司**:
- 建筑工程（中国建筑、中国中铁）
- 造船企业（中国船舶）
- EPC总承包商
- 军工订单企业

**核心分析框架**:
- 在手订单/营收比（Backlog Ratio）
- 订单获取能力评估
- 项目交付周期分析
- 毛利率波动管理

**案例覆盖**: 中国建筑、中国船舶深度分析

### 1.3 强化防跳过机制（多维度验证）⭐⭐⭐

**重大改进**: 从单一时间验证升级为多维度综合验证

**验证维度**:
1. **Token消耗**: 衡量思考深度（step4: 8000+）
2. **内容产出**: 衡量输出质量（step4: 5000+字符）
3. **工具调用**: 衡量行动次数（step4: 18+次搜索）
4. **文件生成**: 衡量工作成果（step4: 9个维度文件）

**通过逻辑**: 硬性要求（内容+文件）必须满足 + 软性要求（Token+工具）互补

**优势**:
- ✅ 不受模型速度影响
- ✅ 不受缓存命中影响
- ✅ 防止"快速生成垃圾"
- ✅ 更关注实际产出

**文档**: `docs/anti-skipping-explained.md`

## Phase 2: 质量提升（重要改进）

### 2.1 新增压力测试模块 ⭐

**文件**: `modules/scoring/stress-testing.md`

**10个极端情景**:
1. 地缘政治: 台海冲突、中美科技脱钩、俄乌式制裁
2. 行业政策: 医药集采扩围、教育式政策、房地产硬着陆
3. 金融风险: 2008式信贷危机、汇率剧烈波动
4. 技术颠覆: 技术路线颠覆、AI对服务业颠覆

**输出**: 生存概率评估、恢复路径分析、风险应对建议

### 2.2 新增季度滚动预测机制 ⭐

**文件**: `modules/analysis/rolling-forecast.md`

**四季更新流程**:
- Q1更新: 输入Q1实际 → 更新全年预测
- Q2更新: 输入H1实际 → 收窄预测区间
- Q3更新: 输入前三季度 → 高精度预测
- Q4闭环: 输入全年实际 → 准确度评估

**准确度追踪**:
- 平均偏差率
- 方向准确率
- 置信区间覆盖率
- 自动生成校准系数

**配套系统**: `core/accuracy_tracker.py`

### 2.3 新增ESG风险调整因子 ⭐

**文件**: `modules/scoring/esg-adjustment.md`

**三维评估**:
- **E(环境)**: 碳成本、环保合规、能源转型 → -5%~+5%
- **S(社会)**: 劳工成本、供应链责任、消费者偏好 → -3%~+3%
- **G(治理)**: 股权质押、关联交易、信息披露 → -4%~+4%

**碳成本计算**: 量化碳排放对企业利润的影响
**风险预警**: 三级预警体系（绿/黄/红/黑）

## Phase 3: 高级功能（战略改进）

### 3.1 新增房地产/REITs驱动型 ⭐

**文件**: `modules/company-types/realestate-driven.md`

**适用公司**:
- 商业地产开发商
- 物流地产企业（普洛斯）
- REITs产品
- 物业管理公司

**核心分析框架**:
- Cap Rate（资本化率）分析: 资产价值 = NOI / Cap Rate
- 租金增长驱动因素
- WALE（加权平均租约期限）
- REITs分红与扩募机制

**案例覆盖**: 普洛斯、中金普洛斯REIT深度分析

### 3.2 新增蒙特卡洛模拟 ⭐

**文件**: `modules/scoring/monte-carlo-simulation.md`

**功能**: 通过10,000次随机模拟，得到营收预测的完整概率分布

**输出**:
- 期望值、中位数、标准差
- P10/P25/P50/P75/P90分位数
- VaR（风险价值）
- 敏感性分析（龙卷风图）

**优势**: 比情景分析更完整的风险画像

### 3.3 新增AI辅助验证工具 ⭐

**文件**: `core/ai_validator.py`

**四维验证**:
1. **完整性**: 必需字段、必需章节
2. **一致性**: CAGR与评分匹配、情景排序
3. **数据质量**: 数据密度、单位一致性
4. **可读性**: 结构层次、段落分布

**输出**: 综合评分（0-100）、质量评级、改进建议

### 3.4 新增跨公司比较分析 ⭐

**文件**: `modules/analysis/cross-company-comparison.md`

**五维比较**:
- 增长能力（30%权重）
- 盈利能力（25%权重）
- 估值水平（20%权重）
- 风险控制（15%权重）
- 质量护城河（10%权重）

**输出**: 行业排名表、雷达图对比、投资建议矩阵

## v2.3.1 (2026-01-16) - 报告验证强化 ⭐ 重要更新

### 核心变更: 自动化报告验证机制

**问题背景**:
- 报告语言有时不是中文（违反框架强制要求）
- 有时只生成JSON文件，遗漏Markdown完整报告
- 报告质量缺乏自动化验证机制
- 缺乏强制阻断机制，验证失败也能继续

**新增功能**:

1. **自动化验证脚本** (validate_report.py):
   - 语言检测: 自动检测中英文比例（必须<30%英文）
   - 文件完整性检查: 必须生成JSON + Markdown两个文件
   - JSON格式验证: 验证必需字段和参数溯源信息
   - Markdown完整性检查: 验证必需章节存在
   - 文件大小检查: JSON>2KB, Markdown>10KB

2. **新增第八步: 报告验证** (skill.md):
   - 强制执行: 报告必须通过验证才能继续
   - 验证失败处理: 禁止进入第九步，禁止更新缓存
   - 明确验证命令: python validate_report.py {公司名}
   - 清晰验证标准: 6项必须全部通过的检查

3. **必需章节清单**:
   - 执行摘要
   - 关键财务指标
   - 双曲线业务分析
   - 情景分析
   - 投资建议
   - 参数溯源

4. **新增检查清单** (checklist.md):
   - 覆盖所有步骤的详细检查项 (第0步到第10步)
   - 品牌矩阵分析专项检查项 (产品驱动型公司)
   - 报告验证专项检查项 (v2.3.1新增)
   - 强制使用要求: 每个步骤完成后必须检查对应项，全部通过才能继续
   - 问题记录表: 记录执行过程中的问题和解决方案

**强制执行规则**:
- ❌ 验证失败 → 禁止继续下一步
- ❌ 语言违规 → 禁止生成报告
- ❌ 文件缺失 → 必须重新生成
- ✅ 全部通过 → 方可进入Step 9

## v2.3.0 (2026-01-13) - 配置加载增强

### 核心变更: 新增配置加载步骤

**问题背景**:
- 缓存目录路径经常配置错误
- 维度文件、输出目录等配置分散各处
- 缺乏统一的配置管理和验证机制

**新增配置加载机制**:

1. **统一配置管理** (core/config.yaml):
   - 目录配置: 缓存根目录、输出目录、搜索结果子目录
   - 路径配置: 模块路径、公司类型路径
   - 维度文件配置: 9个默认维度 + 产品驱动型10个维度
   - 公司类型配置: 9种类型定义及品牌矩阵开关
   - 搜索结果配置: 文件命名模式
   - 语言策略配置: 报告语言、搜索语言映射
   - 格式配置: 日期时间格式
   - 预测配置: 时间窗口、情景概率
   - 评分表配置: CAGR到评分的映射表
   - 缓存新鲜度配置: 数据时效性阈值
   - 搜索策略配置: 搜索次数、深度配置

2. **配置访问接口** (core/config.py):
   - get_cache_base_dir() - 获取缓存根目录
   - get_output_dir() - 获取输出目录
   - get_dimension_files() - 获取维度文件名列表
   - get_dimension_count() - 获取维度文件数量
   - get_company_types() - 获取所有公司类型配置
   - build_company_cache_dir() - 构建公司缓存目录
   - build_search_results_dir() - 构建搜索结果目录
   - validate_config() - 验证配置完整性
   - print_config_validation() - 打印配置摘要

3. **Skill新增第0步** (skill.md):
   - 强制读取 core/config.yaml
   - 强制读取 core/config.py
   - 验证配置加载结果
   - 打印配置摘要信息
   - 验证通过后才能继续分析
